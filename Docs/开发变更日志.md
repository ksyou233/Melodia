# Melodia 开发变更日志

## 文档定位
- 用途：持续记录项目规划的新增、修改、废弃与决策原因。
- 目标：保证任何阶段都能快速回答“改了什么、为什么改、影响到哪里、下一步做什么”。
- 维护频率：每次规划变动后立即更新。

## 更新规则
- 一次变更一条记录，避免合并多件事到同一条。
- 每条记录必须包含：变更内容、原因、影响范围、负责人、状态。
- 状态统一使用：提议中 / 已确认 / 开发中 / 已完成 / 已废弃。
- 若变更影响里程碑，必须同步更新“里程碑影响”章节。

## 风险与跟踪
- 风险：核心循环节奏与建造回报节奏失衡。
  - 观察指标：单局收益、建造等待时间、玩家流失点。
  - 预案：提高基础产出，缩短前期建筑升级时间。
- 风险：内容产能不足导致中期重复。
  - 观察指标：玩家在第2周体验中的重复反馈。
  - 预案：优先做可复用事件模板与任务池。
